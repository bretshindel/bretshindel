from sklearn import neighbors

def plot_decision_regions(X, y, classifier, test_idx = None, resolution = 0.02):
    #setup marker generator and color map
    markers = ('s', 'x', 'o', '^', 'v')
    colors = ('red', 'blue', 'lightgreen', 'pink', 'cyan')
    cmap = ListedColormap(colors[:len(np.unique(y))])
    
    #plot the decision surface
    x1_min, x1_max = X[:, 0].min() - 1, X[:, 0].max() + 1
    x2_min, x2_max = X[:, 1].min() - 1, X[:, 1].max() + 1
    xx1, xx2 = np.meshgrid(np.arange(x1_min, x1_max, resolution),
                          np.arange(x2_min, x2_max, resolution))
    Z = classifier.predict(np.array([xx1.ravel(), xx2.ravel()]).T)
    Z = Z.reshape(xx1.shape)
    plt.contourf(xx1, xx2, Z, alpha=0.4, cmap=cmap)
    plt.xlim(xx1.min(), xx1.max())
    plt.ylim(xx2.min(), xx2.max())
    
    #plot all the samples
    X_Test, y_Test = X[test_idx, :], y[test_idx]
    for idx, cl in enumerate(np.unique(y)):
        plt.scatter(x=X[y == cl, 0], y=X[y == cl, 1],
                   alpha = 0.8, c = cmap(idx),
                   marker=markers[idx], label = cl)
        
    #highlight test samples
    if test_idx:
        X_Test, y_Test = X[test_idx, :], y[test_idx]
        plt.scatter(X_Test[:, 0], X_Test[:, 1], c='',
                   alpha=1, linewidth=1, marker='o',
                   s=55, label='Test Point')

Trying a Different ML Model¶

Looking at the scatter plot of the data, a good modeling choice could be a kernelized Support Vector Machine (SVM) because the points are not linearly seperable. The kernelized SVM is effective at precisely grouping clusters of data into classifications by changing the 'gamma' level.

To solve a nonlinear problem via SVM, we transform the training data into a higher dimensional feature space via mapping function phi(dot) and train a linear SVM model to classify the data in this new feature space. We can apply the same mapping function to transform new, unseen data to classify it using the linear SVM model.

Kernelized SVM is more effective for the job than our KNN classifier because we can set a tighter precision. With KNN=3 the model is underfitting (despite K being fairly low), but lowering K to 1 could result in strong overfitting in most scenarios. The precision from the kernelized SVM allows us to simply expand or contract the decision boundary while keeping a smooth shape, without reaching for outliers. This makes it easier (in this instance) to control the bias-variance tradeoff than KNN does.

K = 89 gives the best accuracy¶

So why did we bother building a validation curve? If a model is too complex for a given training dataset - there are too many degrees of freedom or parameters in this model - the model tends to overfit the training data and does not generalize well to unseen data. It is important to choose the ideal hyperparameters of a machine learning model that will produce the best results. While simply running the model over and over again can for each hyperparameter level can be one method of determing the ideal value, some models can suffer from overfitting as a result of this process because the model gets attuned to the test data. By plotting the model validation accuracies as a function of the level of K, we can visually detect if the model suffers from high variance of high bias.

Our validation score was proven not to be useless (x=37 still has a relatively high accuracy score), and the trends it shows is consistent with our true accuracy plot, but it did not give us the best accuracy after all.
